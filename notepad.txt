PROJECT OVERVIEW
================================================================================
The Student Record Management System is a comprehensive Java application designed 
to manage and maintain student records efficiently. This project demonstrates 
professional software development practices using Object-Oriented Programming (OOP) 
principles and advanced File I/O operations.

Author: Obsinan Muhammed Ahmed
Language: Java
Repository: ObsinanX/Student-Record-Management-System

================================================================================
WHAT THE PROJECT DOES
================================================================================
This system provides a complete solution for managing student information with 
the following capabilities:

CORE FEATURES:
- Add Student Records: Create new student profiles with ID, Name, Department, 
  and GPA
- Search Student by ID: Quickly locate specific student records
- Update Student Information: Modify existing student details
- Delete Student: Remove student records from the system
- Display All Students: View complete list of all managed students
- Generate Student Reports: Create comprehensive reports with statistics
- Create Backup Files: Maintain data integrity through backup functionality
- Display File Properties: View information about stored files

STUDENT DATA STRUCTURE:
Each student record contains:
  * Student ID (Unique identifier)
  * Name (Student's full name)
  * Department (Academic department)
  * GPA (Grade Point Average)

================================================================================
PROJECT STRUCTURE
================================================================================

The system is organized into 4 main classes:

1. Student.java
   Purpose: Represents a student entity
   Responsibilities:
   - Stores student information (ID, Name, Department, GPA)
   - Implements serialization for object persistence
   - Provides getters and setters for data encapsulation

2. FileHandler.java
   Purpose: Manages all file-related operations
   Responsibilities:
   - Text file handling (reading/writing student records)
   - Binary file handling (efficient data storage)
   - Object serialization/deserialization
   - Backup file creation
   - File information display

3. StudentManager.java
   Purpose: Orchestrates student record management
   Responsibilities:
   - Add new student records
   - Search for students by ID
   - Update student information
   - Delete student records
   - Display all students
   - Generate statistical reports
   - Coordinate with FileHandler for persistence

4. Main.java
   Purpose: User interface and program control
   Responsibilities:
   - Menu-driven interface for user interaction
   - Input validation and error handling
   - Coordinate between StudentManager and user
